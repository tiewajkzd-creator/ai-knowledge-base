## 核心架构

**三层流水线设计：**

```
Layer 1: 纯脚本采集（Node.js，不耗 AI token）
   ↓ 8 个 JSON
Layer 2: 8 个 AI Agent 并行分析（各读各的，互不干扰）
   ↓ 8 份独立报告
Layer 3: 主编综合（跨源关联 + 趋势判断 + 行动建议）
   ↓ 1 份最终简报
```

**为什么分三层？**
- 采集与分析解耦，脚本跑采集只要10秒，不花AI token
- 某个源挂了不影响其他源
- 8个分析Agent同时跑互不等待
- 主编能发现跨源信号（同一话题出现在多个平台）

## 数据源配置

**7个免费源：**
- GitHub Trending（curl抓HTML）
- HN / Product Hunt / V2EX
- 微博 / 知乎 / 抖音热搜
- 小红书热点
- 3组RSS：Ben's Bites + MIT科技评论、阮一峰 + 少数派、TechCrunch + 极客公园

**统一CLI接口：**
```bash
node gh-trending.mjs                      # 采集 → JSON
node gh-trending.mjs --extract data.json 0,3  # 提取 → Markdown
```

**扩展新源：** 复制 `_template.mjs` 改名即可，协调器自动发现

## 分析层设计

**独立的分析师Prompt：**
- 每个源对应一个独立的 `.md` prompt文件
- 随时打开微调几行立刻生效，不影响其他源
- 可独立迭代，不断进化

**示例差异：**
- GitHub分析师 - AI Agent框架和前端工具标为P0，游戏和加密货币直接排除
- 小红书分析师 - 从创业者视角深度解读，分析带货机会和话题形态

## 主编综合方法论

**系统化跨源分析：**
1. 实体提取 - 拎出话题、产品、公司、趋势方向
2. 关联矩阵 - 1个源=弱信号，2个源=中信号，3个以上=强信号
3. 信号连续性 - 对比昨天简报，标记新信号、升温/降温
4. 行动分级 - 哪些这周就能动手
5. 选题提取 - 适合写公众号的素材

**铁律：** 不要为了凑关联而硬扯，没有关联就是没有

**输出格式：** 以「30秒速读」开头，三条趋势每条一句证据一句行动

---

## 设计哲学

**核心原则：**
- 采集不耗AI token - 脚本干脏活，AI只做分析
- 分析师互相独立 - 一个源的prompt写烂了不影响其他七个
- 协调器零配置 - 文件系统就是注册表
- 无需数据库 - 不需要npm install、不需要Docker

**本质：** 用文件系统组织的多Agent分析流水线，每个markdown文件是一个Agent的大脑，每个mjs文件是一根数据管道
