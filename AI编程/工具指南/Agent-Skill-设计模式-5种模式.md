## 核心概念

核心思想：**将 agent 拆分为模块**，明确输入、上下文、输出和审核的边界，便于复用、维护和企业场景应用。

## 章节内容

### 00:01 引言
谷歌发布 agent 架构设计方法，强调架构决定 agent 上限，总结了 5 大 agent 技能设计模式。

### 00:46 模块化思考
将 agent 拆成模块，明确输入、上下文、输出和审核的边界，便于复用、维护和企业场景应用。

### 01:23 技能包
完整的 agent 技能包括：
- 角色定义
- 模板
- 参考资料
- 规则说明

执行任务时有明确标准。

### 01:54 Toolwrapper
将明确能力封装成工具，让 agent 调用，提高稳定性，适用于企业智能体。

### 02:31 Generator
适合有**固定结构的内容生成**，模板驱动生成，提高内容稳定性和业务适用性。

### 03:11 Reviewer
模拟真实团队分工：
- 一个 agent 产出
- 另一个 agent 审核

适用于高要求场景，提高内容质量。

### 03:50 Informer
让 agent 在需要时获取上下文，节省上下文，减少干扰，利于系统扩展。

### 04:28 Pipeline
将复杂任务拆成步骤，每步只做一件事，提高：
- 可控性
- 可查性
- 可回溯性

适用于流程明确、结果要求高的任务。

### 05:03 模式选择
根据任务选择模式，考虑因素：
- 结构化产出
- 检查和反馈
- 步骤编排
- 调研工具

### 05:28 模式组合
真正好用的 agent 通常是**多种模式组合**，从单个智能体升级为完整系统。

### 05:56 渐进式披露
随着任务推进，**逐渐给模型需要的信息**，提高模型性能。

## 我的消化

**与 Harness Engineering 的联系：**

| Harness Engineering | ADK Skill 设计模式 |
|---|---|
| 上下文工程 | Informer（按需获取上下文） |
| 架构约束 | Pipeline（步骤拆分） |
| 反馈循环 | Reviewer（生成-审核分离） |
| 垃圾回收 | —（主动遗忘机制） |

**五大模式的本质：** 都是为了解决"AI 自由发挥导致失控"的问题，只是切入角度不同：
- **Toolwrapper** → 限制 AI 能调用什么
- **Generator** → 限制输出的结构
- **Reviewer** → 限制质量下限
- **Informer** → 限制上下文的量
- **Pipeline** → 限制每步的任务范围

**渐进式披露**这个概念最值得注意：不是在开头把所有信息都塞给 AI，而是随着任务推进逐步给。这和 Auto Dream 的思路一致——都是在时间维度上做信息管理。
