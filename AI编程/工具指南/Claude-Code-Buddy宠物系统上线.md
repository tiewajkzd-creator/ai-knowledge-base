## 核心内容

今天Claude Code悄悄上线了宠物模式，输入 `/buddy` 就会孵化出一只专属小宠物。

## Buddy系统架构

### 双层设计：Bones + Soul

**骨架层（Bones）** — 决定宠物长什么样
- 用账户userID + 固定值，由Mulberry32随机生成
- 确定物种、眼睛、帽子、稀有度等所有外观属性
- 绑定userID，所以不管换多少电脑、重装多少次，宠物永远同一只

**灵魂层（Soul）** — 决定宠物性格
- 可以roll，存在本地
- 第一次孵化时，Claude根据物种和属性给宠物取名、写性格描述

### 宠物种类
18种：鸭子、蘑菇、猫头鹰、蜗牛、幽灵、卡皮巴拉等，每种都有独特动画效果。

### 稀有度机制
- 普通 → 稀有 → 史诗 → 传说
- 独立1%概率出Shiny（闪光），与稀有度无关

### 五大属性
调试能力、耐心值、混乱值、智慧值、毒舌值
- 稀有度越高，属性基础值越高
- 每只宠物一个峰值属性 + 一个短板属性

## 重Roll教程（暴力美学）

作者本是"强度党"，孵出了白色普通仙人掌，无法接受，于是研究出了重Roll方法：

**原理漏洞：**
- Claude Code优先用`accountUuid`作为种子（绑定账户，无法伪造）
- 但用`CLAUDE_CODE_OAUTH_TOKEN`环境变量登录时，不会把accountUuid写入~/.claude.json
- 没有accountUuid，/buddy会退而求其次读userID字段
- userID可以随便改！

**步骤：**
1. 用`claude setup-token`搞到OAuth token
2. 删掉~/.claude.json，清掉旧的accountUuid
3. 写一个最精简配置文件（只留hasCompletedOnboarding和theme）
4. 用环境变量方式启动Claude Code
5. 运行buddy-reroll.js脚本，暴力跑5000万次随机userID
6. 找到想要的宠物后，把假userID塞进~/.claude.json
7. 重启Claude Code，输入/buddy

**结果：** 白色仙人掌 → 闪光传说卡皮巴拉！金色传说！！
