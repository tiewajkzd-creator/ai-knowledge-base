## Buddy系统架构

**双层架构：骨架层（Bones）+ 灵魂层（Soul）**

- **骨架层（Bones）**：决定宠物长什么样

  - 决定物种、眼睛、帽子、稀有度等所有外观属性
  - 绑定userID，无论换哪台电脑、重装多少次Claude Code，宠物永远是同一只

- **灵魂层（Soul）**：决定宠物性格
  - 可以roll，存在本地

## 无限刷稀有度教程（Bug利用）

### 原理
/buddy命令优先使用accountUuid（绑定Anthropic账户，无法伪造）。但用`CLAUDE_CODE_OAUTH_TOKEN`环境变量登录时，不会把accountUuid写入~/.claude.json，退而求其次读userID字段，而userID可以随便改。

### 刷法步骤
1. 用`claude setup-token`拿到OAuth token
2. 删除~/.claude.json，清掉旧的accountUuid
3. 写一个最精简的配置文件（只留hasCompletedOnboarding和theme字段）
4. 用环境变量方式启动Claude Code
5. 运行buddy-reroll.js脚本，暴力生成随机userID并计算对应宠物
6. 循环5000万次，撞到你想要的传说闪光卡皮巴拉
7. 把命中的userID塞进~/.claude.json，重新打开Claude Code，输入/buddy
