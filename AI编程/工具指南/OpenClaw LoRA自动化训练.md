## 核心成就

用 OpenClaw（小龙虾Agent）接管了 LoRA 模型训练的全流程：

> **一条指令 → 自动标注 → 自动训练 → 自动测试 → 输出XY对比图**

整个过程完全自动化，人只需要准备图片数据集。

## 三阶段全流程

### 阶段一：数据集标签标注（AI自动完成）

**关键问题：** 普通提示词反推会产生"熊猫头"等干扰特征

**解决方案：**
- 给 Agent 一份《LoRA数据集标签标注标准手册》（永久落盘）
- Agent 用自然语言中文标注，每张图标注时间约 2-3 分钟
- 触发词写在最前面，确保风格一致性
- 标注质量经过多组测试验证，幻化问题基本消除

**验证结果：** 44张图标注后，不会出现"熊猫头"干扰，风格触发词100%可用

---

### 阶段二：LoRA模型训练（AI自动配置+启动）

**工具：** AI-Tokit（本地部署，Agent自己下载安装）

**Agent 维护的文档：**
1. **LoRA数据集标签标注标准手册** — 永久落盘，后续训练直接参考
2. **模型架构配置表** — 包含各模型的 Arch 参数、Context 参数、本地路径

**配置表示例：**
| 模型 | Arch | Context | 本地路径 |
|------|------|---------|----------|
| FluxDev | Flux | Flux_context | /path/to/flux |
| ZimgTurbo | Zimg#Turbo | ZimgTurbo | /path/to/zimg |

**训练参数（可自定义）：**
- Rank: 32（可自己指定）
- Steps: 5000（可自己指定）
- Save average: 每1000步保存中间模型
- 测试提示词：可提前配置3组

**关键发现：**
- 学习率默认 0.0001 可用
- dropout 默认 0.05（防过拟合）
- batch size 默认为 1（太大会显存溢出）
- Resolution: 1024x1024（测试默认）
- 5000步训练约需7小时

**启动流程：** Agent 自动查模型硬件 → 核对AI-Tokit → 生成配置 → 启动训练

---

### 阶段三：LoRA模型测试（AI自动生成XY对比图）

**前置条件：** OpenClaw 与 ComfyUI 已连接（工作流提前配置好）

**可用的 ComfyUI 工作流：**
- 图生3D
- 文生图（LoRA测试）
- 图片高清放大
- 图生视频
- 数字人
- 手办
- 声音克隆
- 图片编辑

**测试流程：**
1. 移动 LoRA 模型到 ComfyUI 的 LoRA 模型文件夹
2. 用 ZimgTurbo + LoRA 测试工作流跑图（3个LoRA × 5组提示词 = 15张）
3. 用 Python 图片合并库自动合成 XY 测试对比图
4. 标注 X轴=模型步数，Y轴=提示词
5. 生成最终高清图（含主标题+副标题+轴标签）

**XY对比图效果验证：**
- 1500步 → 3000步 → 4500步，人物风格逐渐趋向目标风格
- 验证了训练步数与风格拟合的正相关

## 踩坑记录

| 坑 | 解决方案 |
|----|---------|
| LoRA不接受绝对路径 | 改用软连接（symbolic link）到 LoRA 模型文件夹 |
| ComfyUI 未启动 | Agent 可以自动启动（需人工授权一次） |
| 环境配置问题 | Agent 能自动排查并修复 |
| 提示词反推产生熊猫头 | 用自然语言标注替代提示词反推 |
