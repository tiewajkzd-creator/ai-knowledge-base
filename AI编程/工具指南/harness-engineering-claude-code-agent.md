## 核心发现

**🌟 Harness 工程：不是"开发"智能体，而是给它造一个好用的工作环境**

> 模型是司机，Harness 是车。你不需要教司机怎么开车，你只需要造一辆好车。

**Harness = 工具 + 知识 + 上下文管理 + 权限边界**

## 12节课完整体系

### 第一阶段：让它跑起来

| 课节 | 核心 | 解决的问题 |
|------|------|-----------|
| **S01 最小智能体** | 一个循环：大模型不断调用工具，直到自己判断不需要为止 | 智能体能运转了 |
| **S02 工具箱** | 所有允许的工具注册进工具箱（读/写/编辑/执行）；文件工具有围栏，只能操作项目目录 | 能干活，也有边界了 |
| **S03 Todo清单** | 智能体接任务后先列步骤，一步一步打勾；连续3轮没更新Todo，系统自动塞提醒 | 不跑偏了 |

### 第二阶段：让它干得好

| 课节 | 核心 | 解决的问题 |
|------|------|-----------|
| **S04 子智能体** | 父Agent通过task工具创建子Agent；子Agent上下文全新干净，执行完只传回简短摘要 | 能分工，上下文不臃肿了 |
| **S05 技能加载** | Skill以文件夹组织（SKILL.md）；启动时只注入技能名称（几十token）；执行时按需加载完整内容 | 有知识储备，按需取用 |
| **S06 三层压缩** | ① 旧工具返回内容替换成标记 `[Previous: used read_file]`；② 超阈值时自动摘要压缩；③ 智能体可主动调用压缩 | 聊多久都不爆上下文 |

### 第三阶段：让它记得住

| 课节 | 核心 | 解决的问题 |
|------|------|-----------|
| **S07 任务系统** | 每个任务创建独立JSON文件；支持依赖图（blockedBy）；完成后自动解锁下游任务 | 关机了任务还在 |
| **S08 后台任务** | `background_run`丢后台立即返回；结果存入通知队列；每轮自动注入结果给模型 | 不傻等了 |

### 第四阶段：让它带团队

| 课节 | 核心 | 解决的问题 |
|------|------|-----------|
| **S09 组建团队** | 领导Agent创建队友；每个Agent独立线程+独立收件箱文件夹；通过收件箱通信 | 一个人忙不过来，招人 |
| **S10 团队规矩** | 关机：领导发请求+队友收尾完毕才回复同意；审批：队友提交方案，领导批准/拒绝 | 关机要握手，大事要审批 |
| **S11 自治** | 任务看板机制；空闲时每5秒轮询收件箱和看板；有活就自动认领；持续60秒无任务自动关机 | 不用老板盯着，自己找活 |
| **S12 Worktree隔离** | 每个任务绑定独立git worktree目录；各自独立分支，可独立提交/回滚；通过任务ID关联 | 各写各的，别打架 |

## 关键设计思想

### 1️⃣ 循环永远不动
> 从 S01 到 S12，那个循环的代码一行都没变过。变化的是 Harness 的工具、规则和环境。

### 2️⃣ 两层加载（技能）
- **启动层**： Harness 启动时扫描所有 SKILL.md，把技能名+简介注入系统提示（几十token）
- **执行层**：智能体确定需要某技能时，调用 `load_skill` 工具，注入完整内容（几千token）

### 3️⃣ 三层上下文压缩
1. 旧工具返回 → 替换为标记（如 `[Previous: used read_file]`）
2. 超阈值 → 完整对话存磁盘 + 模型摘要替换历史
3. 智能体主动 → 可调用压缩工具

**关键**：历史信息存磁盘，没有真正丢失

### 4️⃣ 任务系统核心
- 任务ID + 三种状态：`pending` / `in_progress` / `completed`
- `blockedBy` 字段管理依赖关系
- 完成任务后，自动查找并解锁下游任务

### 5️⃣ 团队通信机制
- 每个Agent有独立文件夹作为**收件箱**
- 每次调用模型前，自动将收件箱内容注入上下文
- 通过唯一ID实现请求-响应匹配

## 我的消化

**本质洞察**：传统做法是"教模型怎么干活"（写if-else、工作流）；Harness工程的思路是"给模型一个干活的环境"（工具、知识、边界、规则），模型本身负责推理决策。

**和Claw AI Lab的区别**：
- Claw AI Lab = 多Agent科研协作（讨论收敛）
- Harness工程 = 单/多Agent的系统构建方法论（工程化）

**可迁移原则**：
1. **循环不动，工具加到工具箱里** — 核心架构稳定，扩展靠注册机制
2. **按需加载知识** — 启动时给菜单，执行时才上完整食谱
3. **上下文要压缩但不能丢** — 磁盘备份 + 摘要替换

**知识成熟度**：🌱 新鲜观点（单源，待实践验证）
