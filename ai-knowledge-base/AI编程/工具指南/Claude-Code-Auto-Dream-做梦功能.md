## 核心发现

### 1️⃣ Auto Dream 是什么？

Anthropic 在 2026年3月底灰度上线的功能，让 Claude Code 在后台自动整理记忆文件。

**触发条件（需同时满足）：**
- 距离上次整理已过 24 小时
- 中间积累了 ≥5 个对话记录

**触发时机：** 不是在你睡觉时，而是在你下次打开 Claude Code 干活时，后台起一个子代理独自完成整理。

---

### 2️⃣ 四步做梦流程

| 步骤 | 名称 | 做了什么 |
|------|------|----------|
| 1 | Orient 定向 | 读一遍整个记忆目录，搞清楚有哪些文件及关系 |
| 2 | 搜集信号 | 翻对话记录，重点找：纠正 Claude 的地方、让它记住的东西、反复出现的内容、重要决策 |
| 3 | 巩固 | 核心环节——把信号跟现有记忆逐个比对，更新/补充/合并重复/解决矛盾/写对日期/清理无用信息 |
| 4 | 修剪和索引 | 最后过一遍，删除冗余信息，生成干净的 MEMORY.md 索引 |

---

### 3️⃣ Auto Memory 的痛点

Claude Code 会自动记录使用过程中的笔记（存在 `~/.claude/projects/项目名/memory/`），但问题在于：

- **矛盾积累**：「老板喜欢方案A」和「老板改主意了」并存，Claude 不知道该信哪个
- **相对日期失效**：「昨天决定推迟到下周」——过了一个月后鬼知道是哪个昨天
- **记忆爆炸**：913个对话积累的记忆，MEMORY.md 膨胀到 280 行，包含 3 条互相矛盾的 API 错误记录

**Auto Dream 效果：** MEMORY.md 从 280 行 → 142 行，矛盾记录解决，过时框架名更新，所有日期变真实日期。

---

### 4️⃣ Claude Code 的四层记忆架构

| 层级 | 类型 | 说明 |
|------|------|------|
| 第一层 | CLAUDE.md | 手写的指令文件，项目规范/编码标准（最高权限） |
| 第二层 | Auto Memory | 工作过程中 Claude 自己记的笔记 |
| 第三层 | Session Memory | 单次对话内的上下文，对话结束消失 |
| 第四层 | Auto Dream | 后台记忆巩固层，定期清理/整理/优化 |

---

### 5️⃣ 与人类记忆的类比

文章提出了一个有趣的映射：

| 人类大脑 | Claude Code |
|----------|-------------|
| 感官记忆（瞬时，几秒消失） | 对话日志 JSONL（原始记录） |
| 短期记忆（容量7±2，15-30秒） | Auto Memory（随手记，容易乱） |
| 长期记忆（容量无限） | 整理后的 MEMORY.md（结构化） |
| 海马体（临时缓存区） | `memory/` 目录 |
| 睡眠中的记忆巩固 | Auto Dream |

**关键洞察：** 人类睡眠时，海马体把白天的信息重新激活、回放给大脑皮层；Auto Dream 也是类似——从对话日志（海马体）里提取信号，写入结构化的记忆文件（大脑皮层）。

---

### 6️⃣ AI 比人类更强的遗忘能力

人类：忘了就是真忘了。

Claude：**原始对话记录一直保留在本地**，Auto Dream 只是从记忆文件里删掉过时内容，但 JSONL 日志没被碰。

→ 既享受了遗忘的好处（减少噪音），又保留了随时翻旧账的能力。

## 我的消化

**这篇最值得记住的不是 Auto Dream 本身，而是它揭示的方向：**

1. **AI Agent 需要「时间感」** —— 能回顾过去、整理过去、从过去中提炼理解。这标志着 AI 开始有了「历史」。

2. **「做梦」这个命名很妙** —— 把枯燥的记忆整理类比为做梦，既形象又暗示了背后机制的复杂性（不是简单删除，而是重组）。

3. **对 OpenClaw 的启发** —— 我们的 AI大脑+知识库 其实也在做类似的事：采集 → 整理 → 存入知识库。但目前缺少一个「消化-巩固-遗忘」的自动循环机制。或许可以借鉴 Auto Dream 的四步流程？

4. **「选择性遗忘」是 Agent 的必修课** —— 记忆不是越多越好，过时的、矛盾的、冗余的信息反而会降低系统效率。
