# 🗂️ 提示词收录库（Prompt Vault）

**定位：** 收集、整理、复用已验证的优质提示词，作为创作养分长期沉淀。

---

## 📌 核心分类判断：收录 vs 技巧 vs 其他

**这是最关键的决策——我拿到任何内容后，第一步就是判断它属于什么：**

| 类型 | 判断标准 | 去向 |
|------|---------|------|
| **提示词收录** | 含有≥1段**可直接复制使用的完整提示词**，内容目的是"给你一段词" | `提示词收录/[模型]-收录集.md` |
| **提示词技巧** | 讲的是**怎么写提示词的方法/框架/公式**，内容目的是"教你写词" | `提示词技巧/` |
| **混合型** | 既有方法论，又附带了多段可用提示词 | 方法论→技巧，提示词→**单独提取**到收录 |
| **其他** | 工具评测/模型更新/工作流教程/一般性文章 | 对应的其他子目录 |

**判断优先级：看内容主体目的，不看标题。** 一篇标题叫「提示词教程」的文章，如果里面90%是具体提示词原文，那它的本质是收录素材。

## 📥 投喂处理流程（6种输入场景）

### 场景1：直接粘贴提示词文本
```
星爸爸发送：
"a girl standing in cherry blossom rain, soft lighting, film grain..."

或中文：
"赛博朋克风格的少女，霓虹灯光，雨中漫步，电影质感..."
```
**我的处理：**
1. 判断是否为可执行提示词（连续的描述性文本，非对话/提问）
2. 识别目标模型（从参数/语法/关键词/上下文推断；不确定则问一句）
3. 打标签（LLM分析：风格/场景/主题/构图/光影）
4. 按标准格式追加到对应模型收录集
5. 回复确认 + 简要摘要

### 场景2：发送链接
```
星爸爸发送：
"收录一下 https://civitai.com/images/xxxxx"
"这个提示词不错 https://www.xiaohongshu.com/..."
```
**我的处理：**
1. `web_fetch` 抓取页面
2. 提取所有提示词文本（正面/负面/参数）
3. 提取元信息（作者/模型/热度/发布时间）
4. 判断类型：纯提示词展示→收录 / 教程类→技巧 / 其他→对应目录
5. 如果一个页面含多段提示词，逐条提取，批量收录
6. 回复确认 + 收录了几条 + 归入哪个模型

### 场景3：发送截图/图片
```
星爸爸发送：
[图片：某个出图界面/社区帖子的截图]
```
**我的处理：**
1. 视觉理解分析图片内容
2. 提取可见的提示词文本（OCR + 语义理解）
3. 识别界面元素判断模型（ComfyUI/WebUI/MJ/nanobanana界面各不同）
4. 如果能提取完整提示词→走收录流程
5. 如果只能提取部分/模糊→标注为`[部分提取]`，保留原图引用

### 场景4：发送视频
```
星爸爸发送：
[视频：B站/YouTube教程片段]
"这个视频里的提示词帮我收录"
```
**我的处理：**
1. 使用 video-frames skill 抽关键帧
2. 对含提示词的帧做视觉分析提取文字
3. 如有字幕/描述也一并分析
4. 提取出的提示词走收录流程
5. 标注来源为视频截取，附上时间戳

### 场景5：发送文件夹/批量文件
```
星爸爸说：
"这个文件夹里是一堆提示词，帮我整理收录"
路径：~/Downloads/prompts/
内含：xxx.txt, yyy.pdf, zzz.md, ...
```
**我的处理：**
1. 遍历文件夹，读取所有可识别文件（.txt / .md / .pdf / .json / .csv / .xlsx）
2. **逐文件分析内容类型：**
   - 文件内容是纯提示词列表 → 逐条提取，批量收录
   - 文件内容是教程/方法论 → 归入提示词技巧
   - 文件内容是混合型 → 拆分：方法论部分→技巧，提示词部分→收录
   - 文件内容是其他（工具说明/readme等）→ 跳过或归入对应目录
3. 识别每条提示词的目标模型
4. 按模型分组，批量追加到对应收录集
5. 回复汇总报告：
   ```
   📦 批量处理完成：
   - 扫描文件：15个
   - 提取提示词：32条
     ├ nanobanana: 18条
     ├ SD/SDXL: 8条
     ├ MJ: 4条
     └ Flux: 2条
   - 归入提示词技巧：2篇
   - 跳过：1个（README文件）
   ```

### 场景6：对话中被动识别
```
对话过程中：
"我用这个词出图效果很好：a futuristic city..."
"你试试这段：cinematic portrait of..."
```
**我的处理：**
1. 检测到分享意图关键词 + 完整prompt文本
2. **不打断对话**，在回复末尾轻问：「这段提示词要收录吗？」
3. 确认后走收录流程

## 🤖 自动抓取机制

### 抓取策略

| 平台 | 频率 | 抓取目标 | 方法 |
|------|------|---------|------|
| **Civitai** | 每3天 | 过去7天热门图片Top20的提示词 | `web_fetch` 热门页 |
| **LibLib** | 每3天 | 精选/热门提示词 | `web_fetch` |
| **小红书** | 每3天 | 搜索「AI绘画提示词」「Midjourney咒语」等 | xiaohongshu-multimodal skill |
| **Reddit** | 每3天 | r/StableDiffusion r/midjourney 高赞帖 | `web_fetch` |
| **GitHub** | 每周 | awesome-prompts类仓库的更新 | `web_fetch` |
| **nanobanana** | 每3天 | 官方Gallery/社区新作品 | `web_fetch` |

### 抓取流程
```
[Cron定时触发：每3天 09:30，跟在深度采集之后]
         ↓
① 逐平台抓取热门/最新内容
         ↓
② 提取提示词原文 + 元信息（模型/参数/热度/作者）
         ↓
③ 内容类型判断（同投喂场景的判断逻辑）
   - 是具体提示词 → 进入收录流程
   - 是技巧教程 → 走正常知识库采集流程
         ↓
④ 去重（三层）
   - 提示词文本相似度≥85% → 丢弃重复
   - 同作者同模型近似内容 → 保留更优版本
   - 跨平台同源 → 保留最完整版本
         ↓
⑤ 质量评分
   - ≥80分 → 自动收录
   - 60-79分 → 暂存待确认队列
   - <60分 → 丢弃
         ↓
⑥ 格式化 + 打标签 + 追加到对应模型收录集
         ↓
⑦ 同步到GitHub Pages
         ↓
⑧ 推送采集报告：
   「🗂️ 提示词自动采集完成
   新增收录：X条（nanobanana: a, SD: b, MJ: c）
   待确认：Y条（回复编号+保留/丢弃）
   去重丢弃：Z条」
```

### 质量评分维度

| 维度 | 权重 | 评判标准 |
|------|------|---------|
| 互动热度 | 25% | 点赞/收藏/下载数（相对于平台均值） |
| 提示词完整度 | 25% | 正面词+负面词+参数+模型标注齐全≥80分 |
| 独特性 | 20% | 与已收录内容的文本差异度（TF-IDF/语义） |
| 来源可信度 | 15% | 知名作者/认证账号/白名单博主加分 |
| 可复现性 | 15% | 是否标注完整参数、种子、步数等 |

### 待确认队列处理
- 每次采集后推送待确认列表
- 超过24小时未回复 → 自动丢弃
- 下次推送时告知：「⏰ 已自动丢弃X条超时内容」
