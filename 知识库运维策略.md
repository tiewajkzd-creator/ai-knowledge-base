## 🐛 Bug 修复记录

### 2026-04-08 知识库大排查

**问题1：frontmatter 标题被解析为"---"**
- 根因：`generate-data.js` 用 `lines[0]` 直接取第一行，有 frontmatter 时第一行是 `---`
- 修复：升级 `generate-data.js`，优先取 frontmatter `title`，其次取 H1 `# 标题`
- 状态：✅ 已修复

**问题2：微信公众号内容抓取失败**
- 根因：微信公众号有登录墙，`web_fetch` 和 API 都返回空
- 修复流程：
  1. 优先用 `web_fetch` 抓取
  2. 失败则用 `browser screenshot` 截图识别
  3. 截图也失败 → 找博主独立博客/B站/GitHub 等开放平台
  4. 实在拿不到 → 标注"⚠️ 登录墙限制，原文无法抓取，核心观点基于截图识别"
- 状态：✅ 已建立流程

**问题3：文章页点击 404（空白页）**
- 根因：Jekyll 默认把 `AI绘画/工具/xxx.md` 变成 `AI绘画/工具/xxx/index.html`（目录结构）
  SPA 的 `renderCard` 链接指向 `AI绘画/工具/xxx` → 浏览器解析成 `xxx.html` → 文件不存在
- 修复：SPA 的 `renderCard` 链接改为 `AI绘画/工具/xxx/`（带 trailing slash）→ 正确识别为目录 → 加载 index.html
- 状态：✅ 已修复

**问题4：分类错乱（5篇文章放错板块）**
- 典型错误：OpenClaw 工具文放入"AI前沿"、提示词收录放入"AI编程"
- 根因：入库时缺乏分类校验机制
- 修复：建立"分类自动校验"机制（见下方优化项①）
- 状态：⚠️ 待自动化

## ⚙️ 优化项（持续跟进）

### ① 标题 Schema 校验 + 告警
- 触发时机：每次运行 `generate-data.js` 后
- 检查项：
  - [ ] 所有 article 的 title 长度 > 0
  - [ ] 不存在"---"空标题
  - [ ] 不存在无标题条目
- 发现异常 → 立即修复 + 推送通知

### ② 微信公众号双轨抓取
- 优先：`web_fetch` → 解析正文
- 降级：browser screenshot → 截图识别内容
- 最后：搜索博主其他平台账号（独立博客/B站/GitHub）
- 标注：无法抓取时在文件内标注来源局限性

### ③ 分类 LLM 初审
- 入库时 LLM 读取文件前5行
- 判断：当前分类是否合理？
- 不合理 → 自动修正 或 推送星爸爸确认
- 重点检查：Agent系统 vs AI前沿 vs AI编程 的边界模糊内容

### ④ SPA 链接自动验证
- 入库流程末尾：生成 data.json → 抽检最新3篇文章的 URL 格式
- 验证：Jekyll 输出路径 vs SPA 链接格式是否匹配
- 不匹配 → 告警，禁止推送
- 当前 SPA 链接格式：`AI绘画/工作流教程/文章标题/`（带 trailing slash）

### ⑤ 空子分类清理
- 定期清理 data.json 中 `length == 0` 的空子分类
- 避免知识库导航出现无意义空目录
- 当前空分类：
  - Agent系统：实战案例、架构设计
  - AI绘画：AI视频、ComfyUI、工具、工具指南、工具测评、提示词工程、提示词库、问题解决
  - AI编程：教程课程
  - AI前沿：Agent、工具测评
  - 个人成长：（全部为空）

### ⑥ 定期全文抽检
- 频率：每月底
- 内容：抽查各板块文章内容是否完整（>200字）
- 标记：内容过短或只有链接而无正文的条目 → 标 ⚠️ 待补充
