## 核心场景

把"总结文章并生成一张分享图"做成了 OpenClaw 的可复用 Skill：

> **给一个文章链接 → OpenClaw + M2.7 → 直接生成一张可分享的图片**

整个链路自动串联：抓取全文 → 提炼 → 排版 → 生成HTML → 截图

## 5步 Skill 构建法（M2.7 编程流程）

### 第一步：给目标，不给方案

不要告诉它"怎么做"，只说"要什么"。

例如：告诉它使用场景、目标受众、作用是什么。它会自动把这个目标翻译成一个可执行的计划，包括流程步骤和文件结构。

### 第二步：先调研，别闭门造车

**最容易踩的坑：** 直接让它从零开始写。

正确做法：
- 先让它去 GitHub 搜索参考案例
- 找到类似风格后，再确认方向
- 这样能直接站在成熟方案的肩膀上，省去大量试错

### 第三步：方向确认后再动手

和它一起把以下问题确认清楚：
- 目标受众是谁
- 需求方向是什么
- 有哪些现成 Skill 可以直接调用
- 文章结构怎么设计

方向确认后再开始尝试，避免来回改需求。

### 第四步：边跑边补标准

过程中会不断遇到具体问题：
- 标准应该怎么写
- 重点怎么拉
- 信息层次乱怎么办
- 哪个版本更适合长文

M2.7 的稳定性体现在：**不会每轮都把之前做过的东西打散，而是能顺着上下文继续改、继续收。**

### 第五步：跑顺后沉淀成 Skill

把整个链路跑通后，提取哪些是可以复用的，最终沉淀成 Skill。

**核心理念：Skill 不是设计出来的，是跑通之后留下来的形状。**

---

## 三个关键踩坑点

### 坑1：必须用 curl 抓取全文

不能只给标题和摘要就让它开始写。

**正确做法：**
```
用浏览器 curl 把正文完整抓下来 → 再提链和生成页面
```

如果不抓全文，后面页面做得再漂亮也没用。

### 坑2：必须按手机阅读设计

用户最终是在手机端转发和阅读的。

**关键要素：**
- 字号
- 留白
- 排版密度

都得按手机阅读去调，特别是出门时发给 OpenClaw 这个场景很重要。

### 坑3：截图必须全屏

如果截图不全屏，旁边会有白边，质感直接掉。

最后一步截图必须全屏，才能保证最终的分享图质量。

> **Skill 不是设计出来的，是跑通之后留下来的形状。**

M2.7 最重要的能力不是"帮做一个结果"，而是"怎么稳定地产出这个结果"——它能把中间这条链路自己接起来，实现真正的 Agent 自动化。
