## 我的消化与总结

### 🔥 三个核心洞见

**1. CLI > MCP 是大趋势**
作者亲身经历：作为 Superbase 重度用户，竟然不知道自己应该用 CLI 版而非 MCP 版。这说明即便深度用户也难以跟上工具进化速度。核心原因是 2026 年 AI 助手已常驻终端（in-terminal），CLI 工具天然在终端环境运行，无需 MCP 的中间层代理。这对所有工具选型都有启示意义。

**2. Skill Creator = 技能开发的 A/B 测试框架**
这是今天最被低估的工具。传统上我们无法量化 Skill 改进效果，Skill Creator 让你可以：对同一任务对比有/无某 Skill 的效果，对 Skill 的新旧版本做 A/B 测试。这把 Skill 开发从"经验主义"变成"数据驱动"。

**3. Obsidian 的最佳用法是个人助理而非编程**
作者特别强调：Obsidian + Claude Code 的价值在于"个人助理场景"而非编程场景。如果你把 Claude Code 当作个人第二大脑（知识管理、日程记录、笔记整理），Obsidian 的 Markdown 文件图谱可视化就非常有意义。如果是编程场景，用文件树管理代码就够了，Obsidian 的意义不大。

### 💡 可迁移原则

- **选工具原则：** 优先选有 CLI + Skill 双重支持的工具
- **安装任何 CLI 后：** 立即去 GitHub/Documentation 找对应的 Skill
- **工具 vs MCP：** 只要工具支持 CLI，就不要用 MCP 版本
- **NotebookLM：** 适合研究/学习场景，完全免费，是性价比最高的 AI 研究工具
- **Firecrawl vs 网页搜索：** 前者提供结构化、适合 AI 消费的网页数据，后者是实时信息，各有分工
