## 核心观点

### 1. 多Agent协同产品推荐：Accio Work
- 来自**阿里国际站**的多Agent协同产品
- 原为跨境电商商家设计，但多Agent体验做得极好
- **无需邀请码，人人可用**，有免费额度
- 网址：http://www.accio.com/work

### 2. 三省六部制 → Agent协作框架
作者参考了 GitHub 项目 https://github.com/wanikua/danghuangshang，将古代三省六部制映射到Agent架构：

| 古代官制 | Agent职能 |
|---------|----------|
| **中书省** | 出方案（方案设计） |
| **门下省** | 挑毛病审批（审核/质量把控） |
| **尚书省** | 拆任务往下派（任务分解调度） |
| **吏部/户部/礼部/兵部/刑部/工部** | 各管一摊执行（专项执行） |

**核心洞察：** 三省六部制本质解决的是"一群人各干各的但最后不乱"的问题，这个框架搬到Agent协同上，几乎不需要额外设计复杂的协作逻辑，**制度本身就是最好的prompt**。

### 3. 实战创建步骤（Accio Work）

**Step 1：** 登录 Accio Work → 选中智能体 → 新建智能体
**Step 2：** 填写Agent配置（名称/头像/提示词/风格/模型）
**Step 3：** 选工具（官方自带工具集，默认全开）
**Step 4：** 装Skills（从官方skills库选择）
**Step 5：** 小设置（称呼/偏好语言）→ Finish & Launch

**偷懒技巧：**
- 把三省六部的配置截图丢给Claude，让它生成所有Agent配置
- 让Claude用computer use自动完成创建

### 4. 团队组建
- 如法炮制创建所有Agent
- 在侧边栏新增"团队"，把多个Agent组合在一起协同工作

## 消化洞见

> **制度本身就是最好的prompt。**
> 
> 三省六部制之所以能运行1400年，本质是**职责清晰、分工明确、流程自洽**。这和Agent协作的设计哲学高度一致：不需要复杂的协调逻辑，只需要明确的角色定义和流程规范。
> 
> 这个思路可以迁移到任何需要多Agent协作的场景：先定义角色职责（类比古代官制），再设计信息流转顺序（类比奏折审批流程），Agent自然会"各干各的不乱"。
