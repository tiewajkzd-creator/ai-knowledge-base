# 🗂️ 提示词收录库（Prompt Vault）

**定位：** 收集、整理、复用已验证的优质提示词，作为创作养分长期沉淀。

---

## 📌 与「提示词技巧」的区分

| | 提示词收录 | 提示词技巧 |
|--|----------|----------|
| 内容 | 具体的提示词原文 | 方法论/公式/框架 |
| 用法 | 拿来直接用或微调 | 学习怎么写提示词 |
| 例子 | 一段完整的樱花少女提示词 | 「时间轴切片法」公式 |
| 标签 | #提示词收录 | #提示词工程 |

---

## 🧠 自动识别规则（什么内容该进收录库）

### 必须同时满足：
1. **包含完整可执行提示词**（≥30个英文单词 或 ≥50个中文字符的连续prompt文本）
2. **以展示/分享提示词为主要目的**（不是讲方法论，而是给你一段具体的词）
3. **有明确的目标模型**（nanobanana / MJ / SD / Flux / Sora 等）

### 自动识别信号（命中≥2条即归入收录库）：
| 信号 | 权重 | 说明 |
|------|------|------|
| 含 `#提示词收录` / `#PromptVault` / `#提示词素材` 标签 | 直接命中 | 手动标记最优先 |
| 含代码块且内容为英文prompt（非代码） | 高 | 如 ```a girl standing in...``` |
| 含 `--ar` / `--style` / `--v` / `--quality` 等生图参数 | 高 | MJ/nanobanana 专有参数 |
| 含 `negative prompt` / `负面提示词` | 高 | SD系特征 |
| 标题含「提示词分享」「prompt分享」「咒语」「魔法词」 | 中 | 标题意图识别 |
| 来源为 Civitai / LibLib / 小红书提示词贴 / nanobanana社区 | 中 | 平台特征 |
| 内容中提示词原文占比 ≥ 50% | 中 | 内容结构识别 |

### 不应进收录库的（归提示词技巧或其他目录）：
- 讲「怎么写提示词」的教程 → 提示词技巧
- 讲工作流搭建、节点配置 → 工作流教程
- 模型发布/更新公告 → 模型更新

## 📁 内部分类（按目标模型分组）

收录库内部按 **模型** 分文件存储，每个模型一个合集文件：

```
AI绘画/提示词收录/
├── README.md                    ← 本文件（规则说明）
├── nanobanana-收录集.md         ← nanobanana 提示词合集
├── midjourney-收录集.md         ← Midjourney 提示词合集  
├── stable-diffusion-收录集.md   ← SD/SDXL 提示词合集
├── flux-收录集.md               ← Flux 提示词合集
├── video-ai-收录集.md           ← 视频AI提示词(Sora/Seedance/可灵等)
└── other-收录集.md              ← 其他模型提示词
```

**为什么按模型分而不是按风格分？**
- 提示词格式强依赖模型（MJ用`--ar`，SD用negative prompt，nanobanana有自己的语法）
- 同一风格在不同模型的写法完全不同
- 检索时最常见问题是「nanobanana怎么画XX」而不是「日系风格怎么画」

## 📝 每条提示词的标准格式

```markdown
### [编号] [简短标题]

> **模型：** nanobanana | **风格：** 日系/赛博朋克/写实/... | **场景：** 人像/风景/概念/...
> **来源：** [平台 - 作者](链接) | **收录时间：** YYYY-MM-DD
> **评分：** ⭐⭐⭐⭐⭐ | **出图稳定性：** 高/中/低

**提示词原文：**
​```
a girl standing in cherry blossom rain, soft lighting, film grain, 
delicate features, white dress flowing in wind, bokeh background,
masterpiece, best quality --ar 3:4 --quality 2
​```

**负面提示词（如有）：**
​```
blurry, low quality, deformed hands, extra fingers
​```

**参数：** --ar 3:4 | --quality 2 | Steps: 30 | CFG: 7

**效果备注：** 出图稳定，适合春季主题。光影柔和，建议搭配LoRA: xxx

**标签：** `#人像` `#樱花` `#日系` `#柔光` `#电影感`

---
```

每个合集文件内用 `---` 分隔符隔开每条提示词，用 `### [编号]` 方便检索引用。

## 📥 投喂机制（人工喂食流程）

### 场景1：星爸爸直接发提示词文本
```
用户发送：
"这个nanobanana提示词不错：a girl in cyberpunk city, neon lights..."

Agent处理：
① 检测到含完整prompt文本 → 触发收录流程
② 识别目标模型（关键词/参数/用户说明）
③ 自动提取并格式化：
   - 提示词原文
   - 参数（如有）
   - 负面提示词（如有）
④ 自动打标签（LLM分析风格/场景/主题）
⑤ 追加到对应模型的收录集文件末尾
⑥ 执行 auto-classify-and-sync.sh 同步
⑦ 回复确认：「已收录至 nanobanana-收录集 #PV-012，标签：#赛博朋克 #人像 #夜景」
```

### 场景2：星爸爸发链接
```
用户发送：
"收录这个 https://civitai.com/images/xxx"

Agent处理：
① web_fetch 抓取页面内容
② 提取提示词 + 负面提示词 + 参数 + 模型信息
③ 同场景1的④-⑦步骤
```

### 场景3：星爸爸发截图
```
用户发送：
[图片：某个出图界面的截图，含提示词]

Agent处理：
① 识别图片中的文字（OCR/视觉理解）
② 提取提示词部分
③ 同场景1的②-⑦步骤
```

### 场景4：从对话中识别（被动收录）
```
对话中出现：
"我用这个词出图效果很好：xxx..."
"这个咒语试试：xxx..."

Agent处理：
① 检测到分享意图 + 完整prompt
② 询问确认：「检测到提示词，要收录吗？」
③ 确认后执行收录流程
```

## 🤖 自动抓取机制

### 抓取源（按优先级）

| 平台 | 抓取方式 | 频率 | 内容 |
|------|---------|------|------|
| Civitai | web_fetch热门图片页 | 每3天 | 高赞图片的提示词 |
| LibLib | web_fetch精选 | 每3天 | 国内SD社区优质词 |
| 小红书 | xiaohongshu-multimodal skill | 每3天 | 搜索「AI绘画提示词」|
| Reddit r/StableDiffusion | web_fetch | 每3天 | 高赞帖子的提示词 |
| nanobanana社区/文档 | web_fetch | 每3天 | 官方示例和社区分享 |

### 抓取流程
```
定时触发（跟随深度采集，每3天 09:00）
  ↓
① 抓取各平台热门/最新提示词内容
  ↓
② 提取提示词原文 + 元信息（模型/参数/作者）
  ↓
③ 去重检查（与已收录提示词对比）
   - 完全相同 → 丢弃
   - 相似度≥80% → 保留更优版本
  ↓
④ 质量评分（评分维度见下）
   - ≥80分 → 直接收录
   - 60-79分 → 推送确认
   - <60分 → 丢弃
  ↓
⑤ 自动打标签 + 格式化
  ↓
⑥ 追加到对应模型收录集
  ↓
⑦ 同步到GitHub Pages
```

### 提示词质量评分维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 互动热度 | 25% | 点赞/收藏/评论数 |
| 提示词完整度 | 25% | 是否包含完整的正面+负面+参数 |
| 来源可信度 | 20% | 知名作者/平台认证/白名单 |
| 独特性 | 15% | 与已收录内容的差异度 |
| 可复现性 | 15% | 是否标注了足够的复现参数 |

## 🔍 检索规则

收录后的提示词通过以下方式被检索和利用：

1. **按模型检索：** 「nanobanana有什么好的人像提示词？」→ 查 nanobanana-收录集.md
2. **按标签检索：** 「有没有赛博朋克风格的？」→ 全文搜索 `#赛博朋克` 标签
3. **按场景检索：** 「风景类的提示词」→ 搜索场景标签
4. **跨模型参考：** 「MJ这个词能转SD吗？」→ 参考已有词做模型适配
5. **作为知识库养分：** 其他Agent查询提示词相关问题时，从收录库检索返回
