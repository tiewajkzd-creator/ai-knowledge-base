## 核心观点

AI视频分镜最烧钱的环节是生图抽卡。本文提出用Claude Agent + Skill搭建AI分镜师，将影视分镜方法论写进Skill，让AI按专业标准拆解剧情、设计镜头、生成提示词。

---

## 核心方法论：三次翻译

### 翻译链路
1. **文字→画面意图**（导演角色）
   - 把剧本拆成可执行的画面段落
   - 明确每个镜头的叙事目的

2. **画面意图→静态视觉资产**（服化道角色）
   - 设计角色外观、场景布局
   - 生成参考图（九宫格）

3. **画面意图+静态资产→动态视频指令**（分镜师角色）
   - 融合静态资产和动态描述
   - 生成Seedance可理解的提示词

## 五大提示词原则

### 1. 参考图已见原则 ⭐⭐⭐
**核心思想**：参考图负责静态信息，提示词只写变化

**实践方法**：
- 参考图：人物外观、场景布局、光影氛围
- 提示词：动作、运镜、情绪转变、节奏变化
- 效果：信息密度提高一倍，减少冗余描述

**示例对比**：
```
❌ 错误：一个穿白衬衫的男人站在办公室里，阳光从窗户照进来，他转身看向门口
✅ 正确：@角色A 转身看向门口，表情从专注转为警觉
```

### 2. 叙事描述式 ⭐⭐⭐
**核心思想**：用完整段落描述，不用关键词堆叠

**实践方法**：
- 所有维度（动作、运镜、情绪、光影）融合在一条时间线上
- 天然带有节奏感（快慢交替）
- 避免"镜头推进，人物转身，光线变暗"这种列表式

**示例**：
```
@角色A 缓缓转身，目光从窗外收回，落在桌上的文件夹上。
镜头跟随他的视线推进，文件夹上"绝密"二字逐渐清晰。
他伸手翻开，手指微微颤抖，室内光线随着他的动作变暗。
```

### 3. 节拍密度约束
**核心规则**：1拍 ≈ 2.5秒屏幕时间

**实践方法**：
- 单镜头内控制节拍数，避免画面糊、跳
- 多镜头结构可突破限制
- 快节奏场景：1拍1个动作
- 慢节奏场景：2-3拍1个动作

**示例**：
```
5秒镜头 = 2拍
拍1：角色转身（0-2.5秒）
拍2：镜头推进特写（2.5-5秒）
```

### 4. 头尾安全区
**核心规则**：前0.5秒、后0.5秒不放关键内容

**原因**：
- 开头需要静态建立时间
- 结尾需要自然收住
- 避免关键动作被裁切

**实践方法**：
```
0-0.5秒：静态建立（角色站立）
0.5-4.5秒：核心动作（转身、推进）
4.5-5秒：自然收住（定格）
```

### 5. @引用系统化
**核心思想**：素材对应表统一管理

**实践方法**：
- 建立素材对应表（角色、场景、道具）
- 每个@引用必须明确用途
- 场景九宫格需单独裁剪编号

**素材对应表示例**：
```
@角色A：男主角，白衬衫，黑框眼镜
@场景01：办公室，现代简约风格
@道具01：绝密文件夹
```

## Skill设计四组件

### 1. 方法论
定义"什么是对的"
- 分镜原则
- 提示词写法
- 质量标准

### 2. 模板
定义"输出长什么样"
- 节拍拆解模板
- 九宫格模板
- 四宫格模板

### 3. 示例

- 常见错误
- 对比说明

### 4. 审核标准
定义"怎么判断够不够好"
- 评分维度
- 通过标准
- 修改建议

## 实用技巧

### 多镜头结构突破节拍限制
```
场景：追逐戏（10秒）
镜头1（3秒）：角色A奔跑，镜头跟随
镜头2（3秒）：角色B追赶，镜头切换
镜头3（4秒）：两人距离拉近，镜头推进
```

### 参考图裁剪编号规则
```
场景九宫格：scene01_01 ~ scene01_09
角色参考图：char_A_front, char_A_side
道具参考图：prop_01, prop_02
```

### 节奏控制技巧
- 快节奏：短句、多动作、快切换
- 慢节奏：长句、少动作、缓推进
- 节奏转换：用停顿、特写、光影变化

## 关键洞察

1. **信息分层是核心**：静态信息用参考图，动态信息用提示词
2. **叙事优先于技术**：完整叙述比关键词堆叠更有效
3. **系统化管理素材**：@引用系统避免混乱
4. **节拍密度决定质量**：控制信息量避免画面崩坏
5. **头尾安全区必须遵守**：给AI建立和收尾的时间
