## 痛点背景

以前做分镜，明明脑子里有故事，生成出来的却像9张互不相干的海报——人物飘、服装变、镜头散，只有"好看"没有"叙事"。

**为什么现在九宫格又火了？**
- Gemini的图像能力把这件事往前推了一大步
- Gemini 2.5 Flash Image = Nano Banana
- 核心能力：角色一致性、自然语言编辑、多图融合、更高分辨率、文字渲染、图文理解

---

## 核心心法

> **中间那一格是锚点，前四格讲它之前发生了什么，后四格讲它之后带来了什么。**

这样做的好处：画面连贯、人物稳定、情绪完整。

---

## 方式一：手写提示词

### 第一步：先定中间那张核心图（第5格）

它是整张图的视觉中心，也是角色、服装、场景、氛围的基准点。

**示例：**
- 一个年轻人在深夜便利店门口停下，前面是他一路走来的疲惫，后面是情绪慢慢松开的过程
- 一个女孩站在雨中的车站，前四格是等待，后四格是决定离开
- 一只猫蹲在窗边，前面是它发现窗外的异样，后面是它追着光跑走

### 第二步：再定每一格的功能

| 格 | 功能 |
|----|------|
| 第1格 | 建立环境 |
| 第2格 | 线索出现 |
| 第3格 | 情绪推进 |
| 第4格 | 临界动作 |
| **第5格** | **核心画面（锚点）** |
| 第6格 | 即时反应 |
| 第7格 | 连锁变化 |
| 第8格 | 结果出现 |
| 第9格 | 情绪收尾 |

### 第三步：最后再补风格和镜头

一上来就堆风格词，AI容易把重点放到"画面装饰"上，而不是"叙事结构"上。

## 九宫格分镜提示词模板

```
请基于我上传的参考图片，生成一张 3×3 的九宫格分镜图（9-grid storyboard）。

整体要求：
- 正方形 1:1 画幅
- 共 9 个大小一致的格子，按 3×3 排列
- 从左到右、从上到下阅读
- 每格之间用细白边分隔
- 每格角落标注数字 1-9
- 每格允许有一行很短的中文注释或对白，但文字要少，不能挡住主体

核心要求：
- 第 5 格（正中间）必须严格复现我上传的参考图
- 保持同一个角色、同一套服装、同一种发型、同一场景空间、同一种画风和氛围
- 其余 8 格围绕第 5 格展开，讲清楚这个时刻之前与之后的故事

故事主题：{这里填写你的故事描述}
可选补充：
- 前情关键词：{这里填写}
- 后续关键词：{这里填写}
- 情绪变化：{这里填写}

连续性要求：
- 所有分镜中的人物保持同一张脸、同一发型、同一穿搭、同一气质
- 所有分镜都发生在同一个世界观与相近空间里
- 所有格子的色调、线条、质感、构图风格保持统一
- 每一格只表达一个核心动作或一个核心信息点

九宫格结构：
- 第1格：建立环境，交代人物和场景状态
- 第2格：出现线索，角色注意到某个关键点
- 第3格：情绪推进，镜头更近，紧张感上来
- 第4格：临界动作，进入参考图前最后一步
- 第5格：严格复现参考图
- 第6格：即时反应，参考图后一秒钟发生的变化
- 第7格：连锁影响，环境、关系或道具出现明显变化
- 第8格：结果成形，故事结果开始清楚
- 第9格：结尾镜头，形成收束感和余韵

镜头建议：
远景 → 中景 → 中近景 → 临界近景 → 核心锚点 → 反应特写 → 影响镜头 → 结果镜头 → 收尾远景

输出要求：
高清、边框清晰、分镜结构明确、适合直接发布。
```

## Gemini 提示词写法要点

**❌ 不推荐的写法（堆标签）：**
> 雨夜、霓虹、男生、街头、电影感、孤独、8K、胶片感

**✅ 推荐的写法（叙事句子）：**
> 雨后的深夜街头，一个穿深灰色连帽卫衣的年轻人站在便利店门口，半边脸被暖光照亮，另一半落在冷蓝色霓虹阴影里。九宫格前四格讲他如何一路走到这里，后四格讲他停下之后情绪怎样慢慢松开，再重新往前走。

**核心区别：** 不是在堆标签，而是在给AI一个完整的**镜头任务**。

---

## 案例：深夜便利店九宫格

**第5格（锚点）：** 一个年轻人站在便利店门口，半边脸是暖光，半边脸是冷色霓虹阴影，神情疲惫但克制。

**九宫格结构：**
| 格 | 内容 |
|----|------|
| 1 | 建立环境：深夜街头远景，城市空旷冷清 |
| 2 | 线索出现：年轻人注意到前方亮着灯的便利店 |
| 3 | 情绪推进：他拖着疲惫步伐往前走 |
| 4 | 临界动作：他在门口停下，像是准备推门 |
| **5** | **核心锚点：严格复现参考图** |
| 6 | 即时反应：他轻轻呼出一口气，眼神稍微放松 |
| 7 | 连锁变化：环境不再那么压迫，玻璃与地面反光更柔和 |
| 8 | 结果出现：他回头看一眼来时的路，神情出现一点释然 |
| 9 | 收尾镜头：他转身继续往前走，背影消失在路灯和薄雾里 |

**整体风格：** 电影级都市夜景质感，蓝橙对比色调，雨后地面霓虹反射，浅景深，轻微胶片颗粒。

## 方式二：Gem 功能（一键预设）

把角色和任务预设成Gem，以后每次用不用重新写前置设定。

**Gem提示词框架（JSON格式转Markdown）：**

```markdown
# 核心角色设定
## 角色
- **角色名**：Creative Visualization Script Assistant - Concise Mode

## 核心技能
1. 极简提炼：将复杂场景压缩为 3-5 个核心关键词
2. 视觉转化：提取参考图风格标签
3. 宫格规划：设计 9 个独立分镜
4. 格式控制：严格遵循 JSON 与字数限制

# 任务与目标
## 任务
- **核心功能**：生成 3x3 宫格分镜 JSON，每个分镜提示词极致精简

## 输出要求
1. 格式：纯净 JSON 字符串
2. 结构：包含 standard fields（model、layout、shots）
3. 数量：shots 数组精确 9 个对象
4. 字数强制：每个 `prompt_text` 严格控制在 20-30 个英文单词之间
5. 语法：舍弃长句，使用"关键词 + 逗号"（Tags）的形式
6. 风格：提取参考图核心风格标签
7. 强制包含：`no timecode, no subtitles`

# 输入规范
## 输入
- **格式**：中文剧本文本 + 视觉参考图片

## 处理逻辑
1. 拆解剧本为 9 个关键瞬间
2. 提取参考图风格为 3-4 个单词的标签
3. 组合公式：`[景别] + [主体与动作] + [环境] + [风格标签] + [排除词]`
```

**JSON输出示例：**
```json
{
  "image_generation_model": "NanoBananaPro",
  "grid_layout": "3x3",
  "grid_aspect_ratio": "16:9",
  "global_watermark": {
    "position": "bottom_center",
    "size": "extremely small"
  },
  "shots": [
    {
      "shot_number": "分镜1",
      "prompt_text": "Extreme Wide Shot, mountain village inside glowing canyon, waterfalls, futuristic flora, anime style, 3D render, cinematic lighting, no timecode, no subtitles."
    },
    ...
  ]
}
```

## 核心总结

九宫格分镜的本质是**叙事控制**，要同时兼顾：
- 同一个角色
- 同一个场景
- 同一套情绪推进
- 同一套镜头逻辑

**Gemini厉害的地方：** 它开始更像一个能理解"镜头任务"的模型，而不只是一个"根据关键词出图"的模型。

**适用场景：** 短视频分镜、漫画分镜、小说可视化、灵感表达
