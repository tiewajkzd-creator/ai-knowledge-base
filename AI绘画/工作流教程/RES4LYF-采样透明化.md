## 核心观点

RES4LYF 把 AI 生成从「盲盒试错」变成了可实时监测并精准调控的确定性流程。

**为什么原生 KSampler 是盲盒？**
- 同一个提示词，步数 CFG 调高调低，结果大不一样
- 看不懂底层区别，不确定步数边界，不确定 CFG 强度
- 整个采样过程完全断层，看不到噪声演化轨迹

**RES4LYF 的本质：** 打开采样黑箱，让你第一次真正看见整个过程——看到噪声如何消失、调度曲线是否合适、细节在哪一步出问题。

---

## 核心原理

**名称拆解：**
- **RES** = Refined Exponential Sampler（精炼指数采样器）
- **4LYF** = Full Life

**底层技术：Bunkman's 线性代数技巧**
- 显著加速采样收敛，让模型更积极搜索潜空间
- 几乎不额外消耗 CPU/GPU 和显存
- **效果：20 步 ≈ 传统采样器 50 步的画质**

---

## 四大核心功能

### 1️⃣ 采样透明化（可视化反馈）
- 把步数、Shift 值、调度变化画成动态图表
- 实时展示采样降噪的完整轨迹
- **曲线末端掉太猛 → 预判细节会出问题**
- **曲线太平 → 构图和结构难以稳定**
- 配合潜空间状态和预测方向的实时预览，就像给生成过程装了监控

### 2️⃣ 无损链式采样（PKback 背负机制）
- 传统采样器断开后细节丢失（衣服纹理、头发细丝、脸部结构）
- RES4LYF 把历史步骤所有状态信息无损添加到潜空间图像
- **可以按每一步拆分，细节无损传递**
- 可以在采样中途更换模型、提示词、LoRA 或插入区域重绘
- 所有操作都能精确到步进行调整和控制

### 3️⃣ ETA（每步纠错噪声注入）
- 在每步结束时精准注入纠错噪声
- 给采样过程添加自动纠错机制
- 让画面在细节和创造力之间达到完美平衡

### 4️⃣ 原生支持无限区域条件
- 可以同时开启十几个区域
- 每个区域分配独立提示词，互不干扰

---

## 其他功能

- **逆采样：** 把现有图片干净还原成潜空间噪声状态，配合重采样为噪声图和精准修改提供完美控制精度
- **循环迭代节点：** 针对细节增强、人脸保护、高复杂度长提示词，设置极其稳定的循环采样策略
- **手动 CFG 调度：** 逐步降低 CFG 值，有效减少后期过度拟合

## 附录：B站视频完整转写工作流（2026-04-02 验证）

**工具链：** yt-dlp + ffmpeg + whisper.cpp

**Step 1 — 下载音频**
```bash
yt-dlp -f "bestaudio[ext=m4a]" --max-filesize 100M -o "/tmp/视频名.%(ext)s" "B站链接"
```

**Step 2 — 格式转换**
```bash
ffmpeg -i /tmp/视频名.m4a -ar 16000 -ac 1 -c:a pcm_s16le /tmp/视频名.wav
```

**Step 3 — 下载 whisper 模型（如未安装）**
```bash
mkdir -p ~/.whisper
curl -L -o ~/.whisper/ggml-small.bin "https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-small.bin"
```

**Step 4 — 转写**
```bash
whisper-cli -m ~/.whisper/ggml-small.bin -l zh -f /tmp/视频名.wav -otxt -of /tmp/视频名
```

**Step 5 — 整理入库**
- 读取 /tmp/视频名.txt
- 按统一小结模板生成 Markdown
- 执行 auto-classify-and-sync.sh

**清理：** 转写完成后删除临时文件
