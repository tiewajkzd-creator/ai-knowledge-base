## 核心观点

### 1. Skill 是什么？
作者用游戏技能做类比来解释：

> 你的技能1 "火球术" 对待不同怪物有不同的"流程、组合技、什么高级装备发出的、属性"
> 
> 等同于：
> - "流程"「说明书」= SKILL.md
> - "组合技"「引用文档」= references
> - "高级装备"「放置脚本」= scripts
> - "属性"「资源目录」= assets

**一句话总结：Skill = 带说明书的提示词。**

这和 OpenClaw 的 Skill 架构高度一致！

### 2. Lovart Skill 介绍
垂直于**设计 Agent**的技能包平台。

**支持场景：**
- 电商 · Amazon Listing Images（亚马逊产品套图）
- 排版 · Marketing Brochure & Print Layouts（营销宣传册/三折页）
- 电影分镜脚本设定

**使用流程（以电商为例）：**
1. 选择 "完整产品套图" 模板
2. 输入自己的产品描述
3. 上传产品图
4. 记得选择 **Nano Banana Pro** 模型
5. 出图快速且稳定

### 3. 实用技巧
- **排版设计**：利用"移除背景"功能处理不规则banner
- **三折页设计**：直接交给 Skill 技能包完成
- **案例复用**：输入框上方有预设案例可直接修改使用

## 消化洞见

> **Skill 的本质是结构化的领域知识封装。**
>
> Skill = SKILL.md（说明书）+ references（引用文档）+ scripts（执行脚本）+ assets（资源）
>
> 这套结构和 OpenClaw 的 Skills 规范几乎一致。意味着：
> 1. Skill 可以跨平台复用（一个好的设计类 Skill 同时适用于 Lovart / OpenClaw / 其他Agent）
> 2. **"带说明书的提示词"** 这个定义值得写入 AI绘画板块的 Skill 创建指南
> 3. Nano Banana Pro 是 Lovart 平台的模型，和之前你关注的提示词优化方向相关
