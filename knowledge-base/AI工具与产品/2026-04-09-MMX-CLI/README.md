## 摘要

今天我们发布 MMX-CLI，一个面向 AI Agent 的命令行工具。模型能力的消费者，正在从人和开发者扩展到 Agent 自身，而 Agent 最自然的工作方式是执行一条命令、拿到一个结果，过去一段时间，命令行已经成为 Agent 在终端里完成工作的常见形态，我们也希望 MiniMax 的全模态能力可以用同样的方式被 Agent 顺手地用起来。

接入 MMX-CLI 后，Agent 可以在 Claude Code、OpenClaw 等环境中原生调用 MiniMax 最新的编程、视频生成、语音合成、音乐创作等全模态模型，无需适配繁琐接口，也无需额外编写 MCP Server。一个 Agent 可以独立跑通"资料搜集 — 生成文案 — 合成语音旁白 — 配图配乐 — 视频制作"这样的完整自动化工作流。

## 核心亮点

### 1. 输出隔离与纯数据模式
让 Agent 拿到的结果是干净的，不被进度条和彩色字符干扰。进度条、模型状态等人类友好的提示信息划归 stderr，stdout 仅输出干净的文件路径或 JSON 数据。结合 `[--quiet]` 与 `[--output] json`，可彻底切断交互式界面，防止转义字符干扰 Agent 解析。

### 2. 语义化状态码（Exit Code）
失败的时候返回一个数字代号，Agent 不需要读英文报错就知道是哪种错、该不该重试。鉴权失败、参数错误、超时、网络异常都设置了独立的退出码，Agent 仅通过读取状态码即可判断并执行重试逻辑。

### 3. 非阻塞与异步任务控制
参数不全时直接退出而不是傻等输入，长耗时任务可以一键转后台，让 Agent 同时做别的事。MMX CLI 让 Agent 在非交互式环境中，参数缺失直接报错退出，避免任务挂起等待输入；针对长耗时任务，`[--async]` 支持一键开启异步模式，满足 Agent 并行处理多任务的需求。

## 安装方式

仅需两行代码，即可完成 MMX-CLI 的安装和调用：

```bash
npx skills add MiniMax-AI/cli -y -g
npm install -g mmx-cli
```

## 使用示例

```bash
mmx image generate --prompt "产品展示图，白色背景" --out-dir ./assets/ --quiet
```
