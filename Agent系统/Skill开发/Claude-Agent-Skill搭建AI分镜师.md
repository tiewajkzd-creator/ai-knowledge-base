# Claude Agent Skill搭建AI分镜师

## 核心观点

用Claude的Agent + Skill功能搭建AI分镜师，将影视分镜方法论（清晰、简洁、连贯、分层渐进）写进Skill，通过制片人+三个Subagent（分镜师、导演、动画师）实现自动化分镜流程。

---

## 系统架构

### 四层设计

```
流程层：CLAUDE.md（制片人）
  ↓ 控制工作流、指令集、Subagent调度
执行层：Subagents（分镜师、导演、动画师）
  ↓ 分工协作
知识层：Skills（方法论、模板、审核标准）
  ↓ 提供专业能力
文档层：outputs/（结构化存储产物）
```

### 三个Subagent

1. **storyboard-artist（分镜师）**
   - 生成节拍拆解
   - 生成九宫格提示词
   - 生成四宫格提示词

2. **director（导演）**
   - 审核所有产出
   - PASS/FAIL把关
   - 打回修改循环

3. **animator（动画师）**
   - 生成动态提示词
   - 用于视频模型

---

## 四个阶段工作流

### 阶段1：节拍拆解
**目标**：从剧本中提取9个关键剧情点

**输出**：`beat-breakdown-ep01.md`

**导演审核标准**：
- 完整性：覆盖剧情起承转合
- 清晰性：每个锚点描述具体
- 选点合理性：关键转折点不遗漏

### 阶段2：九宫格提示词（Beat Board）
**目标**：确定视觉基准

**输出**：`beat-board-prompt-ep01.md`

**核心要求**：
- 锁定人物外观
- 锁定场景布局
- 锁定光色氛围
- 每格独立但风格统一

**导演审核标准**：
- 清晰度：描述具体可执行
- 覆盖度：9个锚点全覆盖
- 一致性：人物/场景/光色统一

### 阶段3：四宫格提示词（Sequence Board）
**目标**：展开连续镜头序列

**输出**：`sequence-board-prompt-ep01.md`

**核心要求**：
- 继承九宫格的视觉设定
- 展开连续动作
- 保持镜头连贯

**导演审核标准**：
- 动作连贯：镜头间逻辑流畅
- 轴线稳定：不跳轴
- 剪接风险：避免跳切

### 阶段4：动态提示词（Motion Prompt）
**目标**：生成用于视频模型的提示词

**输出**：`motion-prompt-ep01.md`

**核心要求**：
- 简洁：去除冗余
- 聚焦：突出核心动作
- 物理合理：符合运动规律

**导演审核标准**：
- 简洁性：无冗余描述
- 聚焦度：核心动作清晰
- 物理合理性：运动符合常识

## 核心机制

### 1. Agent + Skill解耦
**设计思想**：谁来做 vs 怎么做

- **Agent**：负责调度（制片人、分镜师、导演、动画师）
- **Skill**：负责专业知识（方法论、模板、审核标准）

**优势**：
- 灵活替换方法论
- 不用动Agent配置
- 知识可复用

### 2. 导演审核循环
**机制**：每阶段完成后自动审核

**评分标准**：
- 平均分≥8分：PASS
- 单项≥6分：PASS
- 否则：FAIL，打回修改

**循环直到**：所有维度达标

### 3. Resumable Subagents
**问题**：Claude Code的Subagent默认每次调用都是全新上下文

**解决方案**：
- 首次调用记录agentId到`.agent-state.json`
- 后续调用用resume恢复
- 保持上下文连续

**示例**：
```json
{
  "storyboard-artist": "agent_abc123",
  "director": "agent_def456",
  "animator": "agent_ghi789"
}
```

### 4. 文档驱动
**设计思想**：所有产物写入文件，不只存在对话里

**优势**：
- 随时中断、随时恢复
- 输入`/status`查看进度
- 提示词文件可直接使用

**目录结构**：
```
outputs/
├── beat-breakdown-ep01.md
├── beat-board-prompt-ep01.md
├── sequence-board-prompt-ep01.md
└── motion-prompt-ep01.md
```

## 分镜方法论核心

### 四大原则

1. **清晰**：画面描述具体可执行
2. **简洁**：去除冗余信息
3. **连贯**：镜头之间逻辑流畅
4. **分层渐进**：先锚点后细节

### 分层渐进的逻辑

```
节拍拆解（9个锚点）
  ↓ 锁定关键剧情点
九宫格（视觉基准）
  ↓ 确定人物/场景/光色
四宫格（连续镜头）
  ↓ 展开动作序列
动态提示词（视频指令）
  ↓ 驱动视频模型
```

### 一致性继承机制

**规则**：四宫格必须继承九宫格对应格的设定

**示例**：
```
九宫格第3格：
- 人物：白衬衫男主角
- 场景：现代办公室
- 光色：冷色调，窗外阳光

四宫格展开第3格：
- 镜头1：继承上述设定 + 男主角转身
- 镜头2：继承上述设定 + 镜头推进
- 镜头3：继承上述设定 + 特写文件
- 镜头4：继承上述设定 + 表情变化
```

## 项目配置要点

### 目录结构
```
project/
├── script/              # 剧本源文件
├── outputs/             # 生成产物
├── .agent-state.json    # Agent状态记录
└── .claude/
    ├── CLAUDE.md        # 制片人配置
    ├── agents/          # Subagent配置
    └── skills/          # 技能包
```

### 关键配置细节

1. **Subagent调度**：生成任务由分镜师/动画师执行，审核任务全部由导演执行
2. **模板约束**：所有输出必须按照templates/中的格式
3. **方法论共享**：分镜师和导演共享同一份方法论，确保生成和审核标准一致
4. **多集支持**：通过文件名集数标识（如ep01）区分不同集的产物
5. **项目状态检测**：启动时自动扫描script/和outputs/，检测每集进度并路由到对应阶段

## 模型选择建议

### 能力要求

1. **理解能力**：对分镜方法论的理解深度
2. **执行能力**：遵循模板格式的严格程度
3. **一致性维护**：多阶段保持人物/场景/光色一致

### 推荐配置

- **最佳**：Claude 4.5 Opus（理解深、执行准、一致性强）
- **备选**：Gemini 3 Pro、Kimi K2、GLM 4.7（需测试效果）
- **建议**：选择支持长上下文的模型

### 测试方法

1. 用短剧本测试分镜效果
2. 检查是否遵循方法论标准
3. 评估一致性维护能力
4. 根据结果调整模型或配置

- 时间：约15分钟完成全流程

**产出**：
- 节拍拆解表
- 九宫格分镜图
- 四宫格分镜图
- 动态提示词

**质量**：
- 所有内容基于方法论直接生成
- 未经人工润色
- 每阶段通过导演审核

## 关键洞察

1. **解耦是灵活性的基础**：Agent负责调度，Skill负责知识
2. **审核循环保证质量**：不通过就打回，循环直到达标
3. **分层渐进避免混乱**：先锚点后细节，信息逐层细化
4. **文档驱动支持恢复**：中断后可通过文件恢复进度
5. **一致性继承是关键**：四宫格必须继承九宫格设定
