## 核心观点

**Skills是什么？**
- Skills = 给Claude的"专业知识包"，教它如何完成特定任务
- 本质：Markdown文件 + YAML元数据 + 可选脚本/资源
- 与MCP的关系：MCP提供工具访问能力，Skills提供使用这些工具的专业知识

**Skills vs MCP对比：**

| 维度 | MCP | Skills |
|------|-----|--------|
| 作用 | 提供工具（what） | 教如何用工具（how） |
| 类比 | 给AI一把锤子 | 教AI怎么钉钉子 |
| 内容 | 代码、API | Markdown指令 |
| 适用场景 | 需要新能力 | 需要专业知识/工作流 |

## 基础知识

**文件结构：**
```
my-skill/
├── SKILL.md          # 核心指令文件（必需）
├── references/       # 参考文档（可选）
├── scripts/          # 辅助脚本（可选）
└── assets/           # 资源文件（可选）
```

**Description写法（决定触发率）：**
- ❌ 不好："帮助处理项目"（太笼统）
- ✅ 好："为Notion项目设置完整工作区，包括页面、数据库、模板。当用户说'设置项目'、'新建工作区'或'初始化Notion项目'时使用。"

---

## 五大经典模式

1. **顺序工作流编排** - 多步骤按特定顺序执行
2. **多MCP协调** - 工作流跨多个服务
3. **迭代式优化** - 输出质量需多轮迭代提升
4. **情境感知工具选择** - 根据上下文选不同工具
5. **特定领域智能** - 需要专业知识而非仅工具访问

## 故障排查

**技能不触发：** 修改description字段，加入具体触发短语和使用场景

**技能触发过频：** 加负触发词或描述更具体

**指令没有被遵守：** 
- 保持简洁，多用列表和编号
- 重要内容放最前面，用 ## 重要 标题
- 用具体可操作的指令
