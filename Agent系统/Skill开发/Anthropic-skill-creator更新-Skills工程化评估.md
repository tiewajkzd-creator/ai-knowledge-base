# Anthropic skill-creator更新 - Skills工程化评估

## 核心变化

**从手工作坊到工程化流程**

过去Skills最大的问题：写完之后没法严肃评估。你不知道它有没有让结果更稳，有没有比裸模型更强，模型升级后是不是已经没价值了。

**这次更新的本质：** 不是教你再多写一个skill，而是给你一套方法，判断这个skill到底值不值得存在。

---

## 更新内容

**完整的评估闭环：**

1. **Eval** - 给skill设计测试样本，看它有没有按预期完成任务
2. **Benchmark** - 记录通过率、耗时、token成本
3. **多代理并行测试** - 每个测试在独立上下文里跑，避免串场
4. **Description优化** - 自动调整skill描述，减少误触发和漏触发

**背后机制：** 几类子代理协同（执行、打分、盲评对比、整体模式分析），能比较版本A和版本B谁更好。

---

## 官方案例

**PDF skill优化：**
- 问题：在非可填写表单里容易失败
- 定位：坐标锚点不稳定
- 结果：修完后真正解决

**Description优化效果：**
- 测试对象：Anthropic公开的6个skills
- 结果：5个触发准确率都提升了

## Skills分类

**能力增强型：**
- 把文章拆成结构化笔记、长内容转PPT、大幅提升输出质量
- 核心问题：今天有没有明显强于裸模型？差距不大就该退休

**工作流约束型：**
- 会议纪要、周报、公众号改写、固定格式输出
- 价值在于稳定，不是炫技
- 让模型按你的顺序、标准、格式来

**关键洞察：** 大多数团队最缺的不是偶尔惊艳，而是每次都别跑偏

---

## 实用建议

**更新步骤：**
1. 从GitHub更新最新版skill-creator
2. 挑1-2个最常用的旧skills
3. 跑一遍eval和benchmark
4. 看通过率、耗时、token和对比结果
5. 决定是保留、改description，还是重写

**核心价值：** 把skill当成长期资产来管理，而不是一次性工具
