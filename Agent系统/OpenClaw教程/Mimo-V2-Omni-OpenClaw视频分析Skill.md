## 核心观点

### 1. Mimo-V2-Omni 是什么
- **出品方**：小米（之前在 OpenCompass 上出现过的神秘模型）
- **模型类型**：全模态模型，可理解视频、图像、音频
- **定位**：连接数字世界与物理世界的未来模型
- **与视觉模型的区别**：全模态模型训练数据中音频和视频有逻辑联系，不仅识别声音和画面，还学会通过音画结合的信息进行推理

### 2. 为什么 ChatGPT/Claude 没有真正的视频理解
- 视频放入对话上下文需要发送巨大的 token 头肯（9分钟视频压缩成540p约13万 token）
- 当前方案是通过工具将视频转换成文本来理解（REASONING™工具）
- 这个转换过程会丢失大量信息
- 只有 Gemini 和 Kimi 两个工具商支持视频工具，且只有 Gemini 是完整的全模态支持（图像+音频）

### 3. OpenClaw + Mimo-V2-Omni 的使用方法
- 工具名：**Replay Poly**
- 需要：OpenAI API Key（放入环境变量）
- 调用方式：让 OpenClaw 调用工具，传入视频地址+理解要求
- 关键：一步到位告诉工具最终需要什么，而非让工具默认转写

### 4. 推荐使用场景

| 场景 | 操作 |
|------|------|
| **导演脚本生成** | 上传参考视频，让模型分析并生成专业分镜脚本 |
| **素材分割** | 输出 JSON 格式的片段列表（时间戳+内容描述） |
| **高精度嘴部错误检测** | 识别视频中口误/卡壳的具体时间（精确到毫秒） |
| **素材氛围分类** | 按情绪、画面风格、拍摄方式等维度对素材分类 |
| **情感性瞬间捕捉** | 从海量素材中找出笑容、哭起等决定性瞬间 |
| **转场素材识别** | 自动识别适合做转场的片段 |

### 5. 注意事项
- 免费体验截止：**2026年3月26日**（只剩两天）
- API 调用成本较高，不适合当素材分类器
- 定向免费体验只对特定工具开放，但 Replay Poly 已通过此限制
